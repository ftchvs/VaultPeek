# VaultPeek Icon Review (Rename Pass)

Review of the current app icon against VaultPeek positioning — private,
glanceable personal finance — for the PlaidBar → VaultPeek rename.
Reviewed 2026-06-12.

## What the Current Icon Actually Is

The shipped icon (PR #244) is fully code-generated by
`Scripts/generate-app-icon.sh`, which draws a 1024 px master with CoreGraphics
and packs `Sources/PlaidBar/Resources/AppIcon.icns` via `sips` + `iconutil`:

- Apple-grid 824×824 rounded rectangle (185 px corner radius) on a 1024 canvas.
- Blue vertical gradient: `rgb(0, 102, 230)` at the bottom to
  `rgb(92, 171, 255)` at the top (anchored to `SemanticColors.brand` system
  blue).
- A rising white sparkline (26 px stroke, 85% opacity) across the lower third
  with a 22%-opacity area fill under it.
- A bold white `$` glyph (580 pt system bold, soft drop shadow), centered and
  dominating the plate.

Two clarifications that matter for this review:

1. **The menu bar does not show the app icon.** The menu bar item renders
   monochrome template SF Symbols (`dollarsign.circle` when healthy;
   `exclamationmark.triangle`, `network.slash`, or `exclamationmark.octagon`
   in degraded states — see `Sources/PlaidBar/Views/MenuBarLabel.swift`). The
   app icon's small-size duty is the Dock, Finder, Launchpad, the DMG window,
   notification banners, and System Settings lists.
2. **The README hero image is not the app icon.** README currently hotlinks an
   icons8 money-bag PNG that does not match the shipped `$`/sparkline icon.
   The Linear issue's "money-bag" description refers to that README image, not
   to `AppIcon.icns`.

## Fit With VaultPeek Positioning

| Criterion | Assessment |
|-----------|------------|
| Private/glanceable finance | Partial fit. The `$` + rising sparkline reads as "finance at a glance," which matches the product's core promise. Nothing in the icon communicates "private/local-first" — that story is carried by copy, which is acceptable; encoding privacy as a padlock would pull the icon into the cliché bucket below. |
| Avoids generic banking imagery | Mostly. No bank columns, no piggy bank, no coin stacks. A bare `$` glyph is the most generic element; the sparkline and the utility-blue plate keep it closer to "instrument" than "bank app." |
| Avoids crypto-vault imagery | Yes. No vault door, padlock, shield, safe dial, or coin/token imagery. This also keeps distance from the HashiCorp-Vault connotation noted in [naming-clearance.md](naming-clearance.md). |
| Native macOS feel | Good. Correct Apple icon grid, rounded-rect plate, restrained two-element composition, system-blue anchoring matching `SemanticColors.brand`. |
| Rename consistency | Good. The icon never referenced the PlaidBar wordmark or Plaid's brand (no plaid pattern, no Plaid logo shapes), so the rename does not invalidate it. |

## Small-Size and Contrast Check (Honest)

- **`$` glyph at 16 px:** the glyph is ~57% of canvas height, so roughly 9 px
  tall at the 16×16 size — bold white on a mid-blue plate. Verified by
  unpacking the shipped `AppIcon.icns` (`iconutil -c iconset`) and inspecting
  `icon_16x16.png`: the glyph reads, but the vertical bar of the `$` partially
  dissolves, so at the smallest size it reads closer to a white "S" on blue.
  Still distinct and recognizable as this app; the script's "legible at 16pt"
  claim is true of the glyph shape, slightly generous about the `$` detail.
- **Sparkline at small sizes:** the 26 px stroke scales to ~0.4 px at 16×16 and
  ~0.8 px at 32×32 — effectively invisible. The icon honestly degrades to
  "white `$` on blue" below 64 px, with the area fill surviving only as a
  slightly lighter lower band. That is an acceptable degradation, but the
  sparkline should be treated as decorative at small sizes, not as a carrier
  of meaning.
- **Contrast:** computed against the gradient, white is ~5.2:1 over the bottom
  blue and ~2.4:1 over the top blue; the glyph sits over the middle of the
  gradient at roughly 3–4:1 plus a drop shadow. WCAG text-contrast ratios do
  not formally apply to app icons, and the glyph reads clearly in both light
  and dark Dock contexts. The weakest contrast is the sparkline's upper-right
  segment over the lightest blue — decorative, per above.
- **Menu bar recognizability:** not applicable to the .icns directly (see
  clarification 1). The menu bar identity is the template `dollarsign.circle`
  symbol, which inherits system menu-bar rendering and remains recognizable in
  light/dark and with accessibility contrast settings; degraded states swap
  the glyph shape rather than relying on color, per the repo's
  non-color-meaning rule.

## Recommendation: Keep

Keep the current icon for the VaultPeek rename release.

Rationale: it already matches VaultPeek's positioning better than either
cliché it could have been (generic banking or crypto-vault), it is consistent
with the in-app brand color, it survives small sizes honestly via the `$`
glyph, it carries no PlaidBar/Plaid visual baggage, and it is reproducible
from code with no binary design sources.

One follow-up is recommended regardless (tracked outside this review): replace
the README hero money-bag hotlink with an export of the actual app icon
master, so the repo's public face matches the shipped app and drops the
third-party image dependency.

### What Would Trigger Revisiting

- Public or paid distribution at scale (App Store presence, notarized public
  DMG, marketing site) — worth professional icon work at that point.
- Evidence of user confusion with generic banking apps, or feedback that the
  bare `$` reads as US-only in markets VaultPeek wants to serve.
- A brand decision to visually encode the "peek" metaphor (e.g., a
  distinctive silhouette for Dock recognition among other blue icons).
- Any change to `SemanticColors.brand` away from system blue, which would
  desynchronize the icon plate from the in-app accent.
- macOS icon-style guidance changes (e.g., broader Liquid Glass icon
  treatments) that make the current flat gradient look dated next to system
  apps.

## Menu Bar Icon Styles (AND-377)

Settings → Menu bar exposes a **Menu bar icon** preference that changes only the
healthy/default glyph, via `MenuBarIconStyle` (PlaidBarCore). All styles are
monochrome template SF Symbols so they render natively in light/dark/increased
contrast and stay non-color-only:

| Style | Healthy glyph |
|-------|---------------|
| Dollar (default) | `dollarsign.circle` |
| Minimal | `circle.circle` |
| Chart | `chart.line.uptrend.xyaxis.circle` |

The degraded-state ladder (`exclamationmark.octagon` error, `network.slash`
offline, `exclamationmark.triangle` warning/login/stale) is fixed and overrides
the chosen style, so state is always carried by glyph shape + attention text,
never by the icon style or color.

**Full-color gold coin — intentionally rejected (for now).** A full-color menu
bar icon was considered and not shipped: macOS menu bar items are template
images, so a colored glyph would either be templated back to monochrome or, if
forced via `NSStatusItem`, look non-native and risk contrast failures against
dynamic menu-bar materials/wallpapers — and it must never be the sole status
signal. There is no validated 16pt-legible gold-coin asset, and SF Symbol
monochrome templating is the safe, accessible default. Revisit only with a
prototyped asset that passes a light/dark/high-contrast/wallpaper legibility
pass and keeps the degraded glyph + text overrides.
