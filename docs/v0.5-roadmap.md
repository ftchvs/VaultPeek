# PlaidBar v0.5 Roadmap

PlaidBar v0.5 is the product-readiness release between the polished v0.3/v0.4
demo surface and a trustworthy single-user finance utility.

The release goal is simple: a new user should be able to launch demo mode, check
server readiness, connect sandbox or production intentionally, recover from
common failure states, and understand exactly what local data PlaidBar stores.

## Release Pillars

| Pillar | Outcome | Status |
|--------|---------|--------|
| Onboarding preflight | Sandbox and production setup show server, mode, credential, storage, and linked-item readiness before Plaid Link opens. | In progress |
| Trust-first local data | Settings exposes `~/.plaidbar/`, reveal/copy actions, and confirmation-gated reset semantics. | Shipped before v0.5 branch |
| Safer account management | Removing a Plaid item is confirmation-gated and explains local/Plaid boundaries. | In progress |
| Recovery states | Accounts, transactions, credit, and status surfaces distinguish server offline, no item, no account data, no synced history, and filters returning zero results. | Shipped before v0.5 branch |
| Degraded item handling | Login-required and error states surface reconnect actions from Status, Settings, and account detail surfaces. | Shipped before v0.5 branch |
| Documentation and screenshots | README, PRD, DESIGN, and screenshots match the current product. | Shipped in PR #124 |
| Release integrity | Version metadata, changelog, CI, sandbox smoke, and docs scan are aligned before tagging. | Not started |

## Backlog

### 1. Onboarding Preflight

- Show live readiness rows before opening Plaid Link.
- Include server online/offline, expected environment match, Plaid credential readiness, server storage path, and linked item count.
- Provide a manual "Check Again" action.
- Keep Plaid Link blocked by the existing validation path if readiness is wrong.

### 2. Account Management Safety

- Confirm destructive account removal from Settings > Accounts.
- Explain that removal clears local PlaidBar data for the item but does not
  delete Plaid dashboard records or revoke bank permissions outside PlaidBar.
- Keep reconnect available for `login_required` and error items.

### 3. Screenshot Refresh

- Capture current dashboard states after the v0.5 onboarding and settings
  changes.
- Add screenshots for onboarding preflight, Settings > Accounts, Settings >
  Local Data, Notifications, and Status diagnostics.
- Keep screenshots generated from demo or synthetic sandbox data only.

### 4. Release Metadata

- Update `version.env`, `Info.plist`, `PlaidBarConstants.appVersion`, tests,
  Homebrew formula tag, and changelog for `0.5.0`.
- Confirm release docs still match the actual script and metadata.

### 5. Verification

- `git diff --check`
- `swift build --target PlaidBar --skip-update --disable-keychain`
- Focused tests for touched core/server logic
- `./Scripts/smoke-sandbox.sh` when sandbox credentials are available
- Secret/private-data scan over docs, source, tests, and commands

## Non-Goals

- Hosted backend
- Telemetry
- Multi-user support
- Budgeting workflows
- Investment tracking
- Notarization claims before the distribution path is actually implemented
