## Evaluation method

One consistent method for the whole set: **deterministic pass/fail assertions.**
A scenario passes only if its expected outcome is met exactly.

- **CI backbone (run once, gates the whole set):**
  - Strict-concurrency build — `swift build -Xswiftc -strict-concurrency=complete -Xswiftc -warnings-as-errors --disable-keychain`
  - Full suite — `swift test --skip-update --disable-keychain`
- **Per-capability evidence:** a targeted `swift test --filter <suite>` over the
  suite(s) that exercise that capability (deterministic pass count).
- **Server surface (capability 8):** a live `PlaidBarServer --sandbox` booted with
  **zero credentials** and a throwaway `PLAIDBAR_DATA_DIR`, probed with `curl` for
  the `/health` (open) + `/api/status` (auth-gated, metadata-only) contract, plus
  the `PlaidBarServerTests` contract suite.

## Non-blocking observations (no action taken this pass)

The **`swift test` build** (which is *not* compiled with `-warnings-as-errors`)
emits 99 warning-lines. These do **not** fail the CI gate and are **pre-existing
on `origin/main`** (this run changed no source). They split into:

- **3 distinct source-file concurrency advisories** — accepted by the strict gate
  (region-based isolation under `-strict-concurrency=complete`) but flagged by the
  test build's default checking:
  - `Sources/PlaidBar/App/PlaidBarApp.swift:135` — converting non-Sendable function value to `@MainActor @Sendable (Route) -> Void` may introduce data races
  - `Sources/PlaidBar/App/WindowActivationPolicy.swift:49` — main actor-isolated static property `shared` referenced from a nonisolated context (`init(coordinator: … = .shared)`); "error in the Swift 6 language mode"
  - `Sources/PlaidBar/Views/AppShellView.swift:185` — same non-Sendable `(Route) -> Void` conversion as `PlaidBarApp.swift:135`
- **Test-only redundant-`try` warnings** in `PlaidBarCacheTests`
  (`ReadModelCacheClearRaceTests`, `TransactionCacheClearRaceTests`) — "no calls
  to throwing functions occur within 'try' expression."

These are latent: the package compiles in Swift 5 language mode under the Swift 6
toolchain (per `CLAUDE.md`), so the actor-isolation advisory will harden into an
error if/when the targets move to the Swift 6 language mode. They are out of scope
for this no-failure QA pass and are left for a focused follow-up rather than churn
the tree here. Touching them risks the documented `Sendable`-stripping hazard.
