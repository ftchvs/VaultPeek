# Nightly Capability QA — 2026-07-01

Automated nightly QA pass on VaultPeek (formerly PlaidBar). One realistic scenario
per major capability, run under identical conditions, evaluated with a single
deterministic method.

- **Branch:** `qa/nightly-2026-07-01` (off `main` @ `9791d523`)
- **Toolchain:** Swift 6 toolchain, Swift 5 language mode (as configured in `Package.swift`)
- **Data:** demo fixtures / synthetic only — no real Plaid credentials, tokens, account IDs, or balances

## Quality bar (defined up front)

**100% of scenarios pass with zero strict-concurrency build warnings.**

## Evaluation method (one, consistent)

Deterministic pass/fail assertions. A scenario passes only if its expected
outcome is met exactly, evidenced by:

1. The exact CI gate — `swift build -Xswiftc -strict-concurrency=complete -Xswiftc -warnings-as-errors --disable-keychain` (zero warnings) then `swift test --skip-update --disable-keychain` (all green).
2. Each capability is anchored to the specific Swift Testing suite(s)/tests that assert its behavior; a scenario passes iff those assertions pass within the full run.

The end-to-end demo render (`--render-window-first`) is recorded as
**supplementary** evidence, not part of the pass/fail method.

## Conditions (identical for every scenario)

- Same tree (`qa/nightly-2026-07-01`), same commands, same `--demo` / synthetic data.
- Strict-concurrency build **and** the full `swift test` run once; every scenario is
  evaluated against that single run (no per-scenario condition drift).
