cross-checked against a headless demo render (`--render-window-first`), which

## Notes / known non-defects

- **SwiftPM manifest warning (not a defect, not actionable):** the strict build
  emits one *manifest-level* warning — `dependency 'async-http-client' is not
  used by any target`. This is **not** a strict-concurrency/compiler warning
  (the compile is clean and exits 0 under `-warnings-as-errors`). `async-http-client`
  is an **intentional top-level version pin** at `exact: "1.33.1"`
  (`Package.swift:46-48`): the inline comment documents that 1.34.0 dropped
  `Package@swift-6.0.swift`, so the pin keeps SwiftPM resolution on the
  documented Swift 6.0 baseline. Removing it to silence the warning would
  regress that baseline, so it is deliberately left in place.
- **Server live smoke not run autonomously:** `Scripts/smoke-sandbox.sh` (the
  live `/health` + auth-gated `/api/status` HTTP check) requires real Plaid
  sandbox credentials and is gated off by design when they are absent. Under the
  synthetic-data-only constraint it was not run; capability 8 is instead covered
  by the server unit/integration suites above (status contract is metadata-only,
  auth middleware fails closed, token storage safety), all green.
