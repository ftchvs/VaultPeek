## Investigated, intentionally not changed

`swift build` emits one package-graph diagnostic:

> `warning: 'plaidbar': dependency 'async-http-client' is not used by any target`

This is **not** a strict-concurrency compile warning and does **not** fail
`-warnings-as-errors` (the build completes with exit 0). It is a deliberate,
load-bearing transitive-version pin: `async-http-client` is declared (never
linked/imported by any target — confirmed via `grep -rn "AsyncHTTPClient"
Sources/ Tests/` → no hits) **solely to pin the version Hummingbird resolves
transitively to `exact: "1.33.1"`**, because `1.34.0` dropped
`Package@swift-6.0.swift` and breaks the documented Swift 6.0 baseline
(`Package.swift:46-48`). Removing the dependency to silence the warning would
un-pin it and regress toolchain compatibility — the wrong fix. Logged here as an
accepted toolchain-only exception; no change made.
