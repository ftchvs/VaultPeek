# VaultPeek Nightly Capability QA — 2026-07-02

Automated nightly QA pass. One realistic scenario per major capability, all run
under identical CI-gate conditions, evaluated by deterministic pass/fail
assertions.

- **Branch:** `qa/nightly-capability-2026-07-02` (off `main` @ `10e020b0`)
- **Toolchain gate replicated:** `swift build -Xswiftc -strict-concurrency=complete -Xswiftc -warnings-as-errors --disable-keychain`, then `swift test --skip-update --disable-keychain`
- **Data:** synthetic / demo fixtures only (no Plaid credentials, tokens, or real balances)

## Quality bar (defined up front)

> 100% of scenarios pass with zero strict-concurrency build warnings.

**Evaluation method (one consistent method for the whole set):** deterministic
pass/fail assertions. A scenario passes only if its representative
`PlaidBarCore`/`PlaidBarServer` test suite executes and every assertion in it
passes within a single full `swift test` run under the fixed conditions above.
The GUI demo path is evaluated deterministically via `DemoFixtures*`/read-model
tests rather than launching the menu-bar app headlessly (non-deterministic).
